

## فحص سلامة الامتحان - Exam Integrity Check

### الفكرة
اضافة زر "فحص السلامة" في شريط ادوات تعديل الامتحان (BagrutExamPreview) يقوم بتحليل ذكي شامل للامتحان ويعرض تقرير مفصل بنتائج الفحص مع تصنيف المشاكل حسب الخطورة (حرج / تحذير / معلومة).

### ما الذي يفحصه النظام؟

| الفحص | الخطورة | التفاصيل |
|-------|---------|---------|
| اسئلة فرعية مكررة النص | حرج | اكتشاف نفس النص تحت آباء مختلفين (المشكلة التي حدثت في 2024) |
| ترقيم الاسئلة | حرج | فجوات او تكرار في ارقام الاسئلة الرئيسية |
| خيارات MCQ ناقصة | حرج | اسئلة اختيار من متعدد بدون خيارات |
| اجابة صحيحة مفقودة (MCQ) | تحذير | اسئلة اختيار من متعدد بدون تحديد اجابة صحيحة |
| سؤال multi_part بدون فرعية | تحذير | سؤال متعدد الاجزاء ليس لديه اسئلة فرعية |
| مجموع علامات القسم | تحذير | مجموع علامات الاسئلة لا يساوي علامات القسم المعلنة |
| نص سؤال فارغ | حرج | سؤال بدون نص |
| علامات صفرية | معلومة | سؤال طرفي بـ 0 علامات |
| عدد الاسئلة الاجمالي | معلومة | ملخص عدد الاسئلة والاسئلة الفرعية لكل قسم |

### التصميم البصري

- زر بايقونة Shield/CheckCircle بجانب "وضع التعديل" في الـ header
- عند الضغط يفتح Dialog يعرض نتائج الفحص
- كل نتيجة ملونة حسب الخطورة:
  - احمر للحرج
  - برتقالي للتحذير
  - ازرق للمعلومة
- شريط ملخص في الاعلى: عدد المشاكل الحرجة / التحذيرات / المعلومات
- اذا لم يتم العثور على مشاكل: رسالة نجاح خضراء "الامتحان سليم"

### التفاصيل التقنية

#### ملف جديد: `src/lib/bagrut/examIntegrityCheck.ts`

دالة `runIntegrityCheck(exam: ParsedExam)` تعيد:

```typescript
interface IntegrityIssue {
  level: 'critical' | 'warning' | 'info';
  category: string;
  message: string;
  details?: string; // رقم السؤال او القسم المتاثر
}

interface IntegrityReport {
  issues: IntegrityIssue[];
  summary: {
    critical: number;
    warnings: number;
    info: number;
  };
  passed: boolean; // true اذا لا يوجد critical
}
```

الفحوصات تعمل بالكامل على البيانات المحلية (بدون استعلامات اضافية لقاعدة البيانات) - سريعة وفورية.

#### التعديل في `BagrutExamPreview.tsx`

- اضافة زر "فحص السلامة" في منطقة الـ header (بجانب وضع التعديل واظهار الاجابات)
- اضافة Dialog لعرض نتائج الفحص
- استدعاء `runIntegrityCheck(localExam)` عند الضغط على الزر

### الملفات المتاثرة

1. **ملف جديد:** `src/lib/bagrut/examIntegrityCheck.ts` - منطق الفحص
2. **تعديل:** `src/components/bagrut/BagrutExamPreview.tsx` - الزر والـ Dialog

### ضمانات

- الفحص يعمل على الـ state المحلي فقط - لا يعدل اي بيانات
- لا يؤثر على اي وظيفة موجودة
- يعمل مع كلا نوعي الامتحانات (standard و all_mandatory)
- يكتشف المشكلة الاساسية التي حدثت (تكرار الاسئلة الفرعية بنفس النص تحت آباء مختلفين)
