

# إصلاح تلقائي لأسئلة صح/خطأ عبر الذكاء الاصطناعي

## الفكرة

إنشاء Edge Function جديدة تُرسل جميع أسئلة صح/خطأ المشبوهة (178 سؤال) إلى الذكاء الاصطناعي على دفعات، ليُحدد الإجابة الصحيحة لكل سؤال، ثم تُحدّث قاعدة البيانات تلقائياً مع تقرير بالنتائج.

## الوضع الحالي في قاعدة البيانات

| الفئة | العدد | المشكلة |
|---|---|---|
| إجابتها `choice_false` | 130 | كثير منها خاطئ بسبب الـ bug السابق |
| إجابتها `choice_true` | 32 | غالباً صحيحة |
| خيارات بصيغة `choice_1/choice_2` مع إجابة `choice_true/false` | 16 | معرّف الإجابة غير موجود في الخيارات |
| خيارات بصيغة `choice_1/choice_2` مع إجابة `choice_1/2` | ~5 | صيغة مختلفة تحتاج توحيد |

## الخطة التنفيذية

### 1. إنشاء Edge Function: `fix-true-false-questions`

ملف جديد: `supabase/functions/fix-true-false-questions/index.ts`

**المنطق:**

```text
1. جلب جميع أسئلة صح/خطأ النشطة (178 سؤال)
2. تقسيمها إلى دفعات (20 سؤال لكل دفعة)
3. لكل دفعة، إرسال نصوص الأسئلة فقط إلى الذكاء الاصطناعي مع طلب:
   - تحديد الإجابة الصحيحة (صح أو خطأ) لكل سؤال
   - تقديم تفسير مختصر
4. مقارنة إجابة الذكاء الاصطناعي بالإجابة المخزنة
5. تصنيف النتائج:
   - "confirmed": الإجابة المخزنة صحيحة
   - "corrected": الإجابة المخزنة خاطئة → تم تصحيحها
   - "normalized": الصيغة كانت مكسورة → تم توحيدها
6. تحديث قاعدة البيانات:
   - توحيد جميع الخيارات إلى صيغة قياسية: [{id: "choice_true", text: "صح"}, {id: "choice_false", text: "خطأ"}]
   - تعيين correct_answer بناءً على إجابة الذكاء الاصطناعي
7. إرجاع تقرير مفصل بالنتائج
```

**الذكاء الاصطناعي يُستخدم عبر Tool Calling:**

```text
System Prompt:
"أنت مدقق أسئلة تعليمية. لكل سؤال صح/خطأ، حدد الإجابة الصحيحة بدقة."

Tool Schema:
{
  answers: [{
    question_id: string,
    correct_answer: "صح" | "خطأ",
    explanation: string,
    confidence: "high" | "medium" | "low"
  }]
}
```

### 2. إنشاء صفحة التقرير والتشغيل

ملف جديد: `src/pages/TrueFalseFixPage.tsx`

صفحة بسيطة على route `/tf-fix` تحتوي:

1. **زر "بدء الإصلاح التلقائي"** - يستدعي الـ Edge Function
2. **شريط تقدم** يعرض حالة المعالجة (الدفعة الحالية / الإجمالي)
3. **تقرير النتائج** بعد الانتهاء:
   - عدد الأسئلة المُؤكّدة (إجابتها كانت صحيحة)
   - عدد الأسئلة المُصحّحة (تم تغيير إجابتها)
   - عدد الأسئلة المُوحّدة (تم إصلاح صيغتها)
   - قائمة بالأسئلة المُصحّحة مع الإجابة القديمة والجديدة والتفسير
4. **مؤشر الثقة** لكل سؤال (high/medium/low)
5. **زر "تراجع"** يُعيد الإجابات القديمة (نحفظها قبل التعديل)

### 3. Hook لإدارة العملية

ملف جديد: `src/hooks/useTrueFalseAutoFix.ts`

- استدعاء الـ Edge Function
- تتبع حالة التقدم
- عرض النتائج
- حفظ سجل التغييرات محلياً للتراجع

### 4. ربط الصفحة

- إضافة route `/tf-fix` في `App.tsx`
- إضافة زر "إصلاح تلقائي بالذكاء الاصطناعي" في `ExamBankManagement.tsx`
- إضافة في `config.toml`

## التفاصيل التقنية

### آلية الحماية من الأخطاء

- **نسخة احتياطية**: قبل أي تعديل، يتم حفظ الإجابة القديمة في حقل metadata أو في جدول سجل التغييرات
- **دفعات صغيرة**: 20 سؤال لكل طلب AI لتجنب timeout
- **تأخير بين الدفعات**: 500ms لتجنب rate limiting
- **مؤشر الثقة**: الأسئلة بثقة "low" تُعلّم ولا تُحدّث تلقائياً

### التأثير على البيانات

- يتم تعديل حقلي `choices` و `correct_answer` فقط لأسئلة صح/خطأ
- يتم حفظ نسخة من القيم القديمة قبل التعديل
- لا تأثير على أسئلة الاختيار المتعدد أو الامتحانات المحفوظة
- الامتحانات التي تستخدم هذه الأسئلة ستعتمد الإجابة الجديدة المُصححة تلقائياً (لأنها تشير إلى نفس الـ ID)
