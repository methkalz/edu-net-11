

# ربط جميع الـ 24 لعبة بمحتوى الصف الحادي عشر وتحسين العرض

## الوضع الحالي
- يوجد **24 لعبة مطابقة** في جدول `pair_matching_games`
- فقط **8 ألعاب** مربوطة بمواضيع في جدول `grade11_content_games`
- بطاقة اللعبة تظهر حاليا ضمن قائمة الدروس (نفس المستوى) - لكن النظام يدعم لعبة واحدة فقط لكل موضوع

## المطلوب
1. ربط الـ 16 لعبة المتبقية بالمواضيع المناسبة
2. دعم عرض عدة ألعاب تحت نفس الموضوع
3. جعل بطاقات الألعاب تظهر تحت الدروس بتصميم مميز ومختلف

---

## الخطوة 1: إدراج الألعاب الـ 16 المفقودة في قاعدة البيانات

الربط المقترح بناء على تطابق المحتوى:

| اللعبة | الموضوع المقترح |
|--------|----------------|
| طرق الاتصال بالإنترنت | مقدمة عن الشبكة الواسعة (WAN) |
| طرق الدخول للراوتر/السويتش | طرق الدخول للراوتر/السويتش |
| أوضاع العمل في نظام التشغيل | أوضاع العمل الرئيسية في نظام التشغيل |
| الإعدادات الأولية للأجهزة | إعدادات أولية للراوتر/السويتش |
| إدارة الواجهات والعناوين | إعدادات الواجهات |
| بروتوكولات الشبكة المتوسطة | بروتوكول (NAT) |
| مفاهيم التوجيه الأساسية | التوجيه في الشبكات (Routing) |
| إعدادات الأمان المتوسطة | اعدادات الامان في السوتش |
| قوانين البروتوكولات | البروتوكولات الأساسية |
| حزمة بروتوكولات TCP/IP | نموذج Tcp-Ip |
| بروتوكولات التوجيه المتقدمة | التوجيه الديناميكي (Dynamic Routing) |
| شبكات VLAN والتحويل المتقدم | نظرة عامة على VLAN |
| أمان الشبكات المتقدم | ACL قوائم الوصول |
| نموذج الطبقات OSI وTCP/IP | نموذج OSI |
| تغليف البيانات | تلخيص (قسم النماذج المرجعية) |
| استكشاف الأخطاء وإدارة الأداء | أدوات تشخيص وإدارة الشبكة |

سيتم تنفيذ ذلك عبر أمر `INSERT INTO grade11_content_games` باستخدام أداة إدراج البيانات.

## الخطوة 2: تعديل Hook لدعم عدة ألعاب لكل موضوع

**الملف:** `src/hooks/useLessonGame.ts`

- تغيير الاستعلام من `.maybeSingle()` الى `.select()` لجلب جميع الألعاب المرتبطة بالموضوع
- إرجاع مصفوفة من الألعاب بدلا من لعبة واحدة
- حساب حالة القفل/الإتمام لكل لعبة على حدة
