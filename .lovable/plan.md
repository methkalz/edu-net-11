

## اصلاح توزيع العلامات - حل شامل ونهائي

### هل الخطة تضمن 99% دقة؟

**نعم للمجموع الكلي (100%)** - خوارزمية التطبيع تضمن ان مجموع علامات كل قسم يساوي بالضبط `total_points` المعلن.

**بالنسبة لتوزيع العلامات على الاسئلة الفردية**: الذكاء الاصطناعي مطالب اصلا باستخراج العلامات الفعلية من الـ PDF (السطر 807-822 في الـ prompt). المشكلة الحالية ان الذكاء الاصطناعي احيانا يعين قيم تقديرية (4 لكل سؤال، 2 لكل فرعي) بدلا من القيم الحقيقية. التطبيع يصحح المجموع بالتناسب - اذا كانت القيم الاصلية متساوية فالنتيجة دقيقة 100%، واذا كانت مختلفة فالتناسب يعطي افضل تقريب ممكن آلياً.

**لضمان اقصى دقة**: زر "اصلاح العلامات" يطبق التطبيع ثم يعرض النتيجة للسوبر آدمن في المعاينة - يمكنه مراجعة وتعديل اي علامة يدوياً قبل الحفظ.

### المشكلة الجذرية

دالة `distributePoints` في Edge Function تعمل فقط عندما تكون **جميع** الاسئلة بعلامات صفرية. عندما يعين الذكاء الاصطناعي علامات تقديرية، الدالة لا تطبّع المجموع ليتوافق مع `total_points` المعلن للقسم.

### الحل - 3 مستويات

#### المستوى 1: دالة تطبيع مشتركة (ملف جديد)

**ملف**: `src/lib/bagrut/normalizeExamPoints.ts`

- دالة `normalizeExamPoints(exam)` تعمل على كل قسم:
  1. جمع علامات الاسئلة الطرفية = `currentSum`
  2. اذا `currentSum == 0` او `currentSum == total_points`: تخطي
  3. حساب `factor = section.total_points / currentSum`
  4. ضرب كل علامة طرفية بالمعامل وتقريبها
  5. تعويض الباقي على اخر سؤال لضمان مجموع دقيق
- ترجع الامتحان المعدل مع عدد الاقسام التي تم تصحيحها

#### المستوى 2: زر "اصلاح العلامات" داخل نافذة فحص السلامة

**ملف**: `src/components/bagrut/BagrutExamPreview.tsx`

- تعديل `IntegrityCheckDialog` لقبول callback جديد `onFixPoints`
- اضافة زر "اصلاح تلقائي للعلامات" يظهر فقط عند وجود مشاكل علامات (category === 'مجموع العلامات')
- عند الضغط: استدعاء `normalizeExamPoints` على `localExam`، تحديث الحالة، واعادة تشغيل الفحص تلقائيا
- عرض رسالة نجاح بعد التصحيح (sonner toast)

#### المستوى 3: اصلاح Edge Function للامتحانات المستقبلية

**ملف**: `supabase/functions/parse-bagrut-exam/index.ts`

- اضافة خطوة تطبيع نهائية في نهاية `distributePoints`:
  - بعد كل التوزيعات، لكل قسم: اذا مجموع الطرفية لا يساوي `total_points`، تطبيق نفس خوارزمية التطبيع
  - هذا يمنع المشكلة من الحدوث في اي امتحان مستقبلي

### خوارزمية التطبيع (تفاصيل)

```text
لكل قسم:
  1. leaves = جمع الاسئلة الطرفية
  2. currentSum = مجموع علاماتها
  3. اذا currentSum == 0 او currentSum == section.total_points: تخطي
  4. factor = section.total_points / currentSum
  5. لكل leaf: newPoints = Math.round(points * factor * 2) / 2  (تقريب لاقرب 0.5)
  6. remainder = section.total_points - sum(newPoints)
  7. اضافة remainder على اخر سؤال
```
